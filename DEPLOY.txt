PARTE 2 — PUBLICAR ONLINE (ex.: Vercel)
================================================================================

1) Conta em https://vercel.com → Add New → Project → importe o repositório Git
   (ou faça deploy da pasta com Vercel CLI).

2) Build settings (geralmente detecta sozinho):
   - Build command: npm run build
   - Output directory: dist

3) Environment Variables (no painel do projeto Vercel → Settings → Environment):
   VITE_SUPABASE_URL = https://qghdhewbssfatxmhnwng.supabase.co
   VITE_SUPABASE_ANON_KEY = (cole a anon key do Supabase → Settings → API)

   Marque Production (e Preview se quiser).

4) Deploy. Anote a URL final, ex.: https://gestor-stream.vercel.app

5) Supabase → Authentication → URL Configuration (OBRIGATÓRIO após ter a URL):
   - Site URL: https://SUA-URL-NA-VERCEL.vercel.app
   - Additional Redirect URLs: adicione a mesma URL com /**
     Ex.: https://gestor-stream.vercel.app/**

6) Teste o login nessa URL. Se der tela branca em /admin, confira se o deploy
   usou o vercel.json (rotas do React caem no index.html).
